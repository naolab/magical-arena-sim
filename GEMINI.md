# GEMINI 行動指針

このリポジトリで作業する際は、以下 2 つのドキュメントを**必ず熟読し、内容を遵守**すること：

1. `docs/variant-implementation-guide.md`
2. `docs/effect-classification.md`

## 実行ルール

1. **着手前チェック**  
   変更や新機能の検討を始める前に、上記ドキュメントを読み直し、最新のルール・手順を把握する。特にアクションバリアント追加や特殊効果の処理フローは `variant-implementation-guide.md` に従うこと。

2. **効果タイプの分類**  
   すべての特殊効果は `effect-classification.md` の基準でバフ / デバフ / 中立いずれかに分類し、カウント・クレンジング対象を適切に設定する。分類に迷った場合はドキュメントの判定基準・チェックリストを参照して決める。

3. **差分レビュー前の再確認**  
   PR やコミットをまとめる前に、再度 2 つのドキュメントを参照し、手順・分類・命名規則・UI 反映などに抜けがないかを点検する。必要に応じて該当セクションを引用して自己レビューする。

4. **新規ルールの追記**  
   今後ガイドラインが更新された場合は、この GEMINI.md と元ドキュメントの整合性が崩れないように注意する。変更が必要ならまず 2 つのドキュメントを更新し、その後ここを反映させる。

本ファイルの指示に従い、常に `variant-implementation-guide.md` と `effect-classification.md` を参照しながら作業すること。
