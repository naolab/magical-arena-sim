# UI / デザイン設計メモ（v0.1）

## 画面構成

1. **トップ画面**
   - 中央にスタートボタンのみ配置。
   - バトル仕様の簡易説明またはバトル開始前の注意文を余白上部に小さく表示。
2. **バトル画面**
   - 横長レイアウトを想定（PC ブラウザが基本、スマートフォンは横向き推奨）。
   - 左: プレイヤー状態（HP、ファン率、アンチゲージ、現在の観客指示）。
   - 右: 敵状態と観客構成グラフ。
   - 下部中央: 行動選択 UI（アタック / アピール / ガード）。
   - 下部: ターンログ一覧（スクロール可能）。
3. **リザルト画面**
   - 勝敗結果と最終ステータス（HP、ファン率、アンチレベル）。
   - ハイライトしたターン（決定打やブーイング発生など）を抜粋表示。
   - トップへ戻るボタン（再戦ボタンも検討）。

## レイアウト方針
- レスポンシブ対応は最小限。横幅 1024px 程度の想定ビューを基準に調整。
- モバイル向けは横画面を強く推奨し、縦画面では簡易メッセージを表示して横向きへ誘導。
- グリッドシステムは CSS Grid/Tailwind の `grid-cols-3` などで区切り、主要領域を視覚的に分離。

## カラーパレット（暫定）
- **背景**: #0f1014（ダークグレーに近いブラック）
- **アクセント（敵）**: #d15073（深いマゼンタ）
- **アクセント（プレイヤー）**: #4ea8de（冷たいブルー）
- **アクセント（中立）**: #f4c152（マットなアンバー）
- **文字**: #f3f4f6（明度の高いグレー）、サブテキストは #9ca3af
- **アンチ警告**: Lv1 #f97316、Lv2 #f43f5e、Lv3 #ef4444

## タイポグラフィ
- 見出し: サンセリフ系（例: "Poppins"、"Noto Sans JP"）を検討。
- 本文: 読みやすさ重視で "Noto Sans JP" ベース。行間をやや広めに設定。
- 数値表示は等幅フォント（"Roboto Mono"など）を併用して視認性を確保。

## コンポーネントのトーン & マナー
- 暗い背景にネオン調の光彩やグラデーションを控えめに使用し、アンダーグラウンドな世界観を強調。
- フレームやカードには淡いグロー効果を追加して魔術的な雰囲気を演出。
- 観客指示やアンチレベルの変化はアニメーション（フェード/パルス）で注目を集める。
- ボタンや選択 UI はホバー時に明るく発光する表現を入れる。
